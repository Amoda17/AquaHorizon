Home Page

Technical Description

Html structure

1. The Header Section

  • The header contains the logo, title, and subtitle of the webpage inside div.

2. Navigation-Bar Section

  • This section contains the navigation links in the a tags.

3. Footer Section

  • Page editor links of the all 4 students
  • Styling and Policy Links of the website
  • Copyright Information

4. The video Editor Section

  • The div class="video-container" acts as a wrapper for the video background.
  • The video tag is used as the background video that plays automatically (autoplay), loops indefinitely (loop), and remains muted (muted).
  • The source tag provides the video file and its type (video/mp4).
  • The div class="maintopic" contains the website's title and a welcome message.
  • The a tag with class button1 creates a clickable button to guide users to more information.

5. Topic Divider Section

  • div class="topic-container" serves as a border.
  • div class="topic-background" is styled to create a colored divider according to the website's theme.

6. Overview Section

  • div class="overview" organizes content in a flexbox layout.
  • div class="overview-content" contains headings (h2 tags) and paragraphs (p tags) to describe ocean conservation efforts.
  • div class="overview-image" holds an image for the overview
  • img tag used alt attribute for accessibility.

7. Events Section

  • div class="events" introduces planned activities organized by the poeple who own the website.
  • The div class="row" uses flexbox to neatly arrange events in a structured way.

8. Mission Section

  • div class="mission" creating a transparent box to write the mission.
  • div class="mission-text" use h1 tag and p tag to write the title and the mission.
  • div class="mission-image" holds the image for the right side of the box

9. Section Devider and the Topic Section

  • div class="section-divider" is a div acting as a visual separator for the webiste.
  • div class="topic-container" wraps the topic section in a border.
  • div class="topic-background" add the background for the specific border.
  • div class="topic" contains the heading for the section.

10. Team (Topics) Section

  • div class="team" is the container for all topic thumbnails.
  • div class="thumbnail" this representes each topic as a thumbnail box.
  • img src="img2/marine_polution.jpg" alt="Corals" this image representation with alt text
  • div class="details" contains heading and button for each topic
  • button class="button1" a clickable button for open the specific content pages of all 4 students

Css Structure

1.The logo and the title

  • The logo is positioned on the top left side using margin
  • The title,vertical-lineand the subtitle is placed right after the logo in the given order
  • Used display: flex; to evenly distribute the items horizontally, and the header uses it to align the logo, title, and subtitle.

2. Navigation-Bar and Footer

  • position: fixed;is used in the header and navigation bar to keep them fixed at the top of the page while scrolling.
  • Used flex-wrap: wrap; in navigation-bar and the footer so the items will wrap if necessary when resizing the browzer
  • Used z-index: to ensure it stays above everything
  • Used transition: background-color 0.3s to change the background-color smoothly after 0.3 seconds
  • Used hover Effects to applied to the navigation links and footer

3. Video Background Styling

  • position: absolute;, top: 50%;, left: 50%;, and transform: translate(-50%, -50%); center the video within the container.
  • width: 100%; and height: 100%; ensure full coverage
  • object-fit: cover; makes sure the video fills its container without stretching
  • z-index: -1; places the video behind other elements

4. Topic Divider Styling

  • position: relative; ensures it stays within the document flow.
  • width: 100%; covers the full page width.
  • Applied a backgroud color to sepate the video and the overview content for the border

5. Overview Section Styling

  • display: flex; is used to arrange text and image side by side.
  • align-items: flex-start; ensures content aligns from the top.
  • Appilid the text color to #ffffff so the text is highliged properly.

6. Event Section Styling

  • display: flex; in row class arranges event boxes horizontally.
  • flex-basis: 31%; in event-col class ensures equal distribution within the row.
  • background: #161717; gives each event block a sharp look.
  • border: 1px solid #cce7ff; provides a sharp outline for user vision.

7. Mission Section Styling

  • position: relative; width: 1450px; height: 400px; this defines the mission section's size and position.
  • border: 2px solid rgba(255, 255, 255, .5); border-radius: 20px; add a border and rounded corners
  • backdrop-filter: blur(20px);It adds a glass-like effect to make text easier to read over background images.
  • box-shadow: 0 0 30px rgba(0, 0, 0, 0.5); Adds depth to the section by applying a shadow effect.
  • display: flex; flex-wrap: wrap; Ensures the section adapts to different screen sizes

8. Section Devidor and the Topic Section

  • The divider is styled with a width of 100%, a height of 10px, and a dark blue background (#002855) to visually separate sections.
  • border: 3px solid #84a6cf; this defines a border for the topic box
  • width: 600px; height: 50px; this specifies the size for proper visibility
  • display: flex; justify-content: center; align-items: center; this centers content both horizontally and vertically
  • Add a background-color and a box-shadow

9. Team (Topics) Section Styling

  • display: flex; justify-content: center; flex-wrap: wrap; this ensures the topic thumbnails are flexible and center-aligned
  • box-shadow: 0 10px 20px rgba(0, 0, 0, 0.1); this creates depth and a professional look
  • border-radius: 10px; this applies rounded corners for a modern appearance
  • object-fit: cover; that ensures images properly fill their containers without stretching

Accessibility

1. lang="en" in html tag which allows the people to recognized what language is used in this page

2. HTML Structure focuses on (nav tag), (header tag), and (footer tag) to create a well-organized document that is easily navigable .

3. Accessible Navigation Links:

  • (href attribute) in a tags ensures that each link directs to the appropriate page. For example, the Home link points to home.html.
  • "Page Editor 1" or "Privacy Policy" provides clear context about the content users can expect when clicking

4. Color Choices

  • using colors like black for the text color ensures that text is readable
  • Buttons and links have contrasting hover effects that are visually noticeable

5. Alt text for images

  • The img tag in the header includes an alt attribute, which describes the image for users who rely on screen readers or just in case if the photo doesnt load because of a technical difficulity.

6. Responsiveness

  • (@media Queries)adjust the layout for smaller screens. For example, the nav and header containers change layout when the screen width is smaller than 768px.

Table

Technical Description

HTML Structure

1. Table Structure

  • table tag organizes information into rows and columns
  • thead tag groups the table headers to separate them from the body of the table
  • tbody tag holds the main table data, keeping it separate from the headings
  • th tag is used for column headings
  • td tag holds the actual content, including images and progress bars

2. Images and Icons

  • img tags are used to add relevant images while alt attribute used as an alternative text

3. Progress Bars

  • Used div class="progress"> to creates a container for the progress bar
  • div class="progress-bar high/medium/low" indicates progress levels using CSS classes.

Css Structure

1. Table Structure Styling

  • Used border-collapse: collapse; to remove the space between table borders, making them look like a single border instead of separate ones
  • Used box-shadow: 0px 4px 10px rgba(0, 0, 0, 0.9); gives a shadow effect for the table.
  • Used background-color: #ffffff; for readability
  • Used tr:nth-child(even) { background-color: #f0f8ff; } where it colors every second row of the table with a light background

2. Images and Icons Styling

  • Used width and height to ensure the size of the icon

3. Progress Bars Styling

  • In the class progress width and height provides a visual representation of progress.
  • In the classes high,medium and low width and background-color assigns different colors based on impact levels.

Accessibility

1. Text Contrast and Readability

  • Used high contrast colors like black and white to improve the readability
  • Proper font sizes are used (font-size: 18px; for table content, font-size: 25px; for headings)

2. Alternative Text for Images

  • Each img tag has an alt attribute describing the image

3. Used hover effects

Challenges and Lessons Learned

I ran into a few obstacles while our project was being implemented, but each one offered good teaching moments. Managing the team as a leader was one of the main difficulties. Thankfully, all of my teammates were willing and actively involved, so this was easier than expected. Their collaboration streamlined teamwork, enabling us to assign responsibilities smoothly and maintain focus on our goals.

Creating the website's design by myself created another difficulty for me. In order to make sure the website was both visually appealing and simple to use, I had to be familiar with both HTML and CSS. I had to experiment with various designs, layouts, and formatting methods because I didn't have much experience with web design. I discovered through my own mistakes how to efficiently organize pages while improving our website's overall appearance.

There was also a learning curve in finding a balance between practicality and appeal. It required major planning to make sure that access was simple while keeping the design neat and interesting. In addition, I had to make sure the website was responsive and available on various devices, which called for more edits and improvements.

All things thought out, these challenges have improved my ability to solve problems, increase my knowledge of web design, and see the value of teamwork. Every challenge turned into a teaching moment, and by solving them, I increased my confidence in my technical and leadership abilities.

Compliance

When creating my web pages, I ensured they followed the necessary guidelines for accessibility.

  • I used proper HTML structure with (nav, header, footer) tags to make the website easier to navigate. I also included alt text for images so that visually impaired users can understand the content. Additionally, I used high-contrast colors and readable fonts to improve visibility of my webpage.

  • Images, videos, and references used in my project are either created by me or sourced from platforms that allow proper usage. I have credited resources like iStock, Vecteezy, and W3Schools in the references section.

3. Data Privacy
