Profile Page

Technical Description

HTML Structure:

The User Profile Page is built using various HTML elements to create a clear and organized layout. The main container (<div class="userP">) holds all profile related content. While the <h1> tag displays the page title, <h2> and <h3> tags organize subheadings. "Bio," "Skills," "Goals," and "Experience are some of them." The <p> tag presents descriptive content, such as the user's biography, ensuring readability and structure.

The profile section is created using <div class="profile-container">, which contains a nested <div class="profile-info"> that holds the user's profile picture (<img>), name, and a short personal quote. Personal details such as occupation, age, course, and year of study, are organized using <ul> and <li> elements, ensuring a clear and structured display.

With using <div class="skills">, and each skill section is been created with each skill is inside <div class="skill">. Skill names are listed using an unordered list (<ul>), while proficiency levels are visually displayed With progress bars (<div class="progress">). In addition, thses bars follow inline style to shape their appearence.

The "Experience" section is placed inside a <div> with the class "experience." An <h3> tag is used to introduce the section, and an unordered list (<ul>) organizes the experiences. For additional details, nested lists are used within each experience item.

To structure the content in the "Un Goals" section, which uses a <div> container with the class "un-goals". Each goal has been displayed with an <img> tag, which includes an alt attribute for accessibility, that help screen readers to describe the images.

The volunteering section contain a calendar (<div class="calendar">), which organized using a grid layout to display the days of the month. Moreover, event details such as title, date, time, location, and reminder settings, are presented in event cards (<div class="event-card">),that provides a clear and organized view of upcoming events.

The contact section (<div class="contact">) contains a form with <input> fields for the user's name and email, and a <textarea> for the message. The form submission is managed using the mailto: attribute, enabling direct email communication.

CSS Techniques:

Layout and Positioning: The layout is organized using flexbox and grid to ensure a clean, well-aligned design. The .profile-container uses display: flex; with justify-content: center; to horizontally align the profile details. The .calendar section uses display: grid; with grid-template-columns: repeat(7, 1fr); to create a layout where each column evenly represents a day of the week. This ensures that the days are arranged neatly across the calendar.

Background and Visual Design: Sections such as .skills, .bio, .goals, .experience, and .volunteering have a clean white background with rounded edges(border-radius: 10px;) and soft shadows (box-shadow: 2px 2px 10px rgba(0, 0, 0, 0.1);) to create a polished look. The .profile-info section stands out with a light blue background (background-color: #E0F7FA;), setting it apart from the rest of the content. To enhance user interaction, hover effects are applied to multiple sections, changing the background color to #caf0f8;. The .progress bars visually represent skill levels by adjusting the width dynamically (width: X%;).

Text Styling: The typography is set with font-family: Arial, sans-serif; to ensure a clean and easy-to-read appearance. Headings are centered using text-align: center;, creating a balanced look. Additionally, the content within .profile-info is styled with line-height: 2; to improve readability by making the text more spaced out and easier to follow.

Buttons and Interactivity: The event card includes small icons next to the event details (<img> inside .event-detail), arranged using display: flex; to ensure the text aligns properly with the images. The contact form features a fixed-width button to maintain a consistent look and feel across the page, ensuring that interactive elements are visually uniform.

Accessibility

Alt Text for Images: All images, such as the profile picture (<img src="Img4/Avata.png" alt="profile picture">) and UN goals (<img src="Img4/UN14.png" alt="Goal 1">), include descriptive alt attributes to give context and ensure accessibility for users with visual impairments. This helps users understand the content and purpose of the images.

Semantic HTML Elements: Headings (<h1>, <h2>, <h3>) establish a clear content hierarchy, assisting screen readers. Lists (<ul>, <li>) organize information effectively.

Hover and Focus States: Hover effects highlight clickable sections, while focus states on buttons enhance usability for assistive technology users.

Sitemap Page

Technical Description

HTML Structure:

Semantic HTML elements like <h2> for section headings to create a clear content structure. It also includes <svg> to visually display the sitemap, making it easier to understand the website’s layout.

Navigation is implemented using tags, allowing users to access different sections of the website efficiently.

The page uses SVG elements such as <rect>, <text>, and <line> to create a visual sitemap. The <line> elements are styled with arrow markers to indicate the connections between different sections. This visual approach provides a clear, graphical representation of the website's structure, making it easier for users to understand the site’s organization.

CSS Techniques:

Background and Layout: The .sitemap class uses background-image: url('Img4/sm.jpg'); along with background-size: cover; and background-attachment: fixed; to create an attractive background that remains in place as users scroll. This approach enhances the layout by providing a consistent and immersive visual experience throughout the page.

Styling and Interaction: The .box4 and .home-box classes both include the properties stroke: white;, cursor: pointer;, and transition: fill 0.3s;, which make these elements interactive. When users hover over these elements, their colors change, providing visual feedback that enhances the overall user experience.

Typography and Visibility: The .sitemap text class uses font-size: 24px; and font-weight: bold; to make the text easy to read. The text-anchor: middle; centers the text, while pointer-events: none; ensures that the text doesn’t interfere with clickable elements, like the interactive SVG boxes, enhancing the overall user experience.

Accessibility

Semantic HTML: The use of semantic HTML elements like <h2>, <a>, and <svg> improves accessibility for screen readers. These elements help create a clear structure for the sitemap, making it easier for assistive technol

Text Clarity: The use of bold font styles and larger text sizes ensures that the text is easy to read. In addition, the color changes on hover such as the .box4:hover color change, provide visual cues, making interactive elements more accessible for users with low vision or color

SVG Sitemap Navigation: The visual SVG-based sitemap, created using <svg>, <rect>, <text>, and <line>, offers an alternative navigation method for users who may struggle with traditional menus. The <line> elements include arrow markers (marker-end: url(#arrowhead);), providing clear visual connections between sections. This structured design improves navigation, helping users understand section relationships and find content more efficiently.

Content Page

Technical Description

HTML Structure:

The code uses several crucial HTML elements need to organize the page's content. These include <div>, <h1>, <p>, and <a> tags, which assist both users and browsers easily to get the idea of the page structure, especially to screen readers, those who have visual impairment.

The <h1>, <h2>, and <p> tags define the headings and paragraphs that has been making it easy to follow the content with a clear structure.

Lastly, the <a> tags that are used for links, allowing users to interact with the page and navigate to other sections. For instance, the 'Go Back' button uses an <a> tag to scroll back to the top of the page.

CSS Techniques:

Layout and Positioning: The page layout is created using flexbox to position elements clearly and organised. The .container is set up using a flexbox with a vertical layout, which means the items are arranged one on top of the other. In addition, paragraphs (.paragragh) are set to display as flex, and are aligned in a row by using align-items: center, allowing text and images to sit side by side. Absolute positioning is used to place elements exactly where they are needed on the page. For example, the .nav-section is positioned with a specific distance from the left (250px), placing the heading in a fixed spot. Inside the .nav-section, the .nav-container is positioned to make sure the navigation buttons are aligned correctly.

Text Styling: Different font sizes and styles are used for headings like h1 and h2 to make the content easy to read and create a clear visual structure. Text alignment is adjusted by centering the main title and navigation links, while paragraphs are aligned to be justified, making them easier to read. The color property ensures the text stands out and is easily read against various backgrounds.

Backgrounds and Visual Design: Backgrounds and visual design use images and gradients to make the page more appealing. The .nav-section has a background image (background-image: url('Img4/bcpic.jpeg')) with background-size: cover, which ensures the image fits nicely across the container without getting stretched. The .paragragh uses a gradient background (linear-gradient(#4f4080, #9553ff)) with two colors to make it more visually interesting. Some sections, like the text in .nav-section p, use a semi-transparent white background (rgba(255, 255, 255, 0.8)) to help the text stand out and be easier to read over the image.

Buttons and Interactivity: The "go back" button (.back-button) is set to position: fixed, which keeps it visible at the bottom-left of the page, even when the user scrolls.

The properties like background-image, background-size, and background-position are used to add images to sections and ensure they fit properly. flex, align-items, and justify-content help arrange the content flexibly, ensuring everything is aligned correctly on the page. The properties text-align, font-size, font-weight, and color control how the text appears, making it easy to read and well-organized. The positioning properties position: absolute and position: fixed allow elements to be placed in specific locations on the page, no matter what the rest of the layout looks like. Finally, the hover effect has help users see which element they are click over by changing its color to making it easier to identify and interact with.

Accessibility

Alt Text for Images: The alt attribute is added to all images (e.g., <img src="asidification.jpg" alt="Ocean Acidification">). This describes the image for users with visual impairments who use screen readers, helping them understand what it represents.

Semantic HTML: Using semantic HTML tags like <h1>, <h2>, and <p> helps screen readers understand the page layout, which making it easier to access the content for users with disabilities. Headings like <h1> for the main title and <h2> for subheadings create a clear structure, which helps screen reader users navigate the page more easily.

Navigation and Links: The navigation buttons inside the .nav-container provide useful internal links. For instance, <a href="#Ocean-Acidification" class="nav-button" >Ocean Acidification</a> which making easy to move the specific section on the same page. This is especially helpful for users who rely on keyboard navigation or assistive technologies.

Challenges and Lessons Learned

After adding the header and footer, which were created by Student 2, to the web pages, I encountered some problems.The bodies of the two pages collapsed. To resolve this, I placed all my code (which had been between the header and footer) inside another class. Instead of calling the body and h1 for styling, I applied the class. This solved the issue.

When I implemented the sitemap, I tried using text-align: center; to align the names of the website pages, but it did not function. After researching, I found that text-align does not work on SVG elements. According to w3Schools, the text-anchor attribute can be used instead.

Compliance

I used both semantic and non-semantic HTML with proper contrast between text and background to enhance readability. To avoid syntax errors, the HTML and CSS of the webpages I am responsible for pass W3C validation. There is no harmful information present. In accordance with Janet’s security policies, there is no room for unauthorized tracking scripts.
